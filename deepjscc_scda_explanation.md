# DeepJSCC, SCDA, and the DJSCC Directory — Explained

> All information below is sourced directly from the paper:
> **"Semantic Importance-Aware Communications With Semantic Correction Using Large Language Models"**

---

## 📌 Context: Why These Systems Exist

The paper proposes a novel system called **ULSC (Understanding-Level Semantic Communications)** that transmits *natural language descriptions* of images (not the raw image pixels/features), using LLMs to handle errors. To prove ULSC is better, the authors benchmark it against two existing systems: **DeepJSCC** and **SCDA**.

---

## 1️⃣ DeepJSCC — Deep Joint Source-Channel Coding

### What is it?
**DeepJSCC** stands for **Deep Joint Source-Channel Coding**. It is a classical, well-known **feature-level semantic communication system** that transmits image features directly over a communication channel using Convolutional Neural Networks (CNNs).

### How does it work?
| Stage | Description |
|---|---|
| **Encoder (Transmitter)** | A CNN encodes the source image into a compact latent feature representation |
| **Channel Transmission** | The compact feature vector is transmitted directly over the wireless/network channel |
| **Decoder (Receiver)** | Another CNN decodes the received signal back to reconstruct the original image |

### Purpose in the Paper (as a Baseline)
- Represents the traditional, **pixel-level** approach to semantic communications
- Used to show that directly transmitting CNN features is **more vulnerable** to:
  - **Frame/packet loss** (PSNR and image similarity drop significantly at higher loss rates)
  - **Privacy attacks** (an attacker who intercepts the features can reconstruct the original image — high Attacker-PSNR means the attack was successful)
- Tested with **two packaging strategies**: features packed into **7 frames** and **14 frames**, matching the 7–14 word range of natural language captions generated by the proposed system

---

## 2️⃣ SCDA — Semantic Communication with Dynamic Adaptation

### What is it?
**SCDA** (Semantic Communication with Dynamic Adaptation) is a newer **deep-learning-based semantic communication system** that is specifically designed to handle **dynamic/unseen datasets** without needing to know the specific task of the receiver in advance.

### How does it work?
- Like DeepJSCC, SCDA also operates at the **feature level** — it encodes and transmits visual features directly
- Its key advantage over DeepJSCC is **adaptability**: it doesn't require task-specific fine-tuning at the receiver side
- It is still fundamentally a **feature-level** system, transmitting encoded visual features over the channel

### Purpose in the Paper (as a Baseline)
- Represents the **state-of-the-art** in adaptive semantic communications at feature level
- Also benchmarked at **7 frames and 14 frames** packaging strategies
- Like DeepJSCC, it suffers from:
  - Degraded **image similarity** as the frame loss rate increases
  - Vulnerability to **model inversion attacks** (attacker can reconstruct images from intercepted features)

## 3️⃣ The `DJSCC/` Directory in Your Project

### What is it?
The `d:\ULSC-main\DJSCC\` directory is the **implementation of the DeepJSCC baseline** used for comparison experiments in the paper. It is your local code that trains and tests the DeepJSCC model.

### Key Files and Their Purpose

| File | Purpose |
|---|---|
| [`jscc.py`](file:///d:/ULSC-main/DJSCC/jscc.py) | **Training script** — trains the DeepJSCC Autoencoder on image data using MSE loss and Adam optimizer |
| [`modules.py`](file:///d:/ULSC-main/DJSCC/modules.py) | **Model architecture** — defines the `Encoder`, `Decoder`, `Autoencoder` CNNs and the `block_dropout()` function that simulates frame/packet loss |
| [`test.py`](file:///d:/ULSC-main/DJSCC/test.py) | **Testing script** — evaluates the trained model at various frame loss rates (0.001 → 0.1), saves PSNR and similarity results to Excel files |
| [`similarity.py`](file:///d:/ULSC-main/DJSCC/similarity.py) | Computes **Image Similarity** using the CLIP model (as per Equation 18 in the paper) |
| [`psnr.py`](file:///d:/ULSC-main/DJSCC/psnr.py) | Computes **PSNR** between original and reconstructed images (for Attacker-PSNR metric) |
| `model/`, `7zhen/`, `14zhen/` | Saved model weights for different training configurations |
| `*.xlsx` result files | Output data (PSNR, SM) for each loss rate — used to generate the comparison graphs |

### File Naming Convention in `DJSCC/`

| Prefix | Meaning |
|---|---|
| `7PSNR_*` / `7sm_*` | DeepJSCC trained at error rate **0.01**, features packaged into **7 frames** |
| `14psnr_*` / `14sm_*` | DeepJSCC trained at error rate **0.01**, features packaged into **14 frames** |
| `NO7*` / `NO14*` | DeepJSCC trained at error rate **0** (no error training), 7 or 14 frames |
| `Attacker_7*` / `Attacker_14*` | SCDA results — 7 or 14 frames |
| `Attacker_NO7*` / `Attacker_NO14*` | SCDA trained with no error — 7 or 14 frames |

## 📊 How They Are Compared in the Paper

The paper evaluates all methods on:
1. **Image Similarity** (CLIP cosine similarity, 0–1) — higher is better
2. **Attacker-PSNR** (dB) — **lower is better** (means attacker cannot reconstruct the image well = better privacy)

The frame loss rates tested are: `0.001, 0.004, 0.007, 0.01, 0.04, 0.07, 0.1`

The proposed **ULSC** system outperforms both DeepJSCC and SCDA because:
- It transmits **text descriptions** (not raw features), which are harder for attackers to exploit
- It uses **LLM-based semantic error correction** to recover lost frames
- It applies **semantic importance-aware prioritization** to transmit the most critical words first
